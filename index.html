hispark-rs

Bare-metal Rust + QEMU for HiSilicon RISC-V SoCs — develop without the hardware.

A from-scratch embedded ecosystem for the HiSilicon WS63 (Wi-Fi 6 + BLE + SLE/NearLink) and BS2X / BS21 (BLE 5.4 + SLE) RISC-V chips: peripheral-access crates, a safe HAL, a runtime, a custom Rust toolchain, full-system QEMU machine models, and probe-rs debug support — built and CI-tested entirely in emulation.

The stack

          ws63-svd / bs2x-svd   CMSIS-SVD register descriptions
                   │  svd2rust
                   ▼
          ws63-pac / bs2x-pac   peripheral-access crates (raw RegisterBlocks)
                   │
                   ▼
          hisi-riscv-hal        safe embedded-hal 1.0 drivers + async/embassy
                   │
 hisi-riscv-rt ────┤            startup · linker scripts · interrupt vectors
                   ▼
          ws63-examples         blinky · uart_hello · dma · spi · i2c · embassy · net_ping …
                   │  cargo build  (riscv32imfc, custom `ws63` toolchain)
                   ▼
 hisi-riscv-qemu  ◀──  run with no board:  -M ws63 / -M bs21 / -M bs22 / -M bs20
 probe-rs (fork)  ◀──  on-chip debug: RISC-V Debug Module behind a CoreSight DAP

Status

  • Full QEMU bring-up — blinky + uart_hello and the whole peripheral set boot and self-validate on -M ws63 and the BS2X family.
  • BS2X multi-chip — bs21 / bs22 / bs20 machines; SPI, GADC, I²C, KEYSCAN, QDEC, RTC, TRNG, WDT, DMA, PDM, USB all exercised.
  • probe-rs WS63 support (software-complete) — RISC-V-over-CoreSight DTM, vendor debug sequence, flash-algorithm crate.
  • Hardware-in-the-loop — scaffolding ready; awaiting a board to validate clock/timing.
  • Connectivity (Wi-Fi / BLE / SLE) — the north star; deferred behind HIL (the radio is a closed analog/PHY blob).

Quick start

# clone the monorepo (submodules = the whole stack)
git clone --recursive https://github.com/hispark-rs/hisi-riscv-rs
cd hisi-riscv-rs

# install the custom toolchain (see ws63-rust-toolchain), then build for riscv32imfc
cargo build

# run it with no hardware (see hisi-riscv-qemu)
qemu-system-riscv32 -M ws63 -nographic -kernel target/.../uart_hello
